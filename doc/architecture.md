# 远程小车 — 代码架构与调用链

## 1. 硬件平台

| 项目 | 型号/参数 |
|------|-----------|
| MCU | STC89C52RC (兼容 8052, 8位) |
| 晶振 | 11.0592 MHz (标准 51 波特率晶振) |
| ROM | 8KB (IROM 0x0000–0x1FFF) |
| RAM | 256B IRAM + 256B XRAM |
| WiFi | ESP8266 (AT 指令集, UART 透传) |
| 电机驱动 | L298N 型双 H 桥 (P1.2–P1.7) |
| 显示 | 6 位共阴数码管 (74HC595 驱动) |
| 外设 | LED×2 (P1.0, P1.1), 蜂鸣器 (P2.3), 按键 (P3.4) |

## 2. 文件结构

```
main.c       — 入口 + Timer0 ISR + UART ISR + 主循环
display.c/h  — 数码管扫描 (Tab[] 字模, DispBuf[6], T1 ISR)
PWM.c/h      — 电机软件 PWM (moto() 状态机, 4 种运动模式)
usart.c/h    — UART 初始化 + AT 指令发送 + ESP8266 配网
delay.c/h    — 软件延时 (delay_ms / delay_us)
hardware.h   — 引脚宏定义 (led1, led2, buzzer, key1)
fix.h        — VS Code/clangd 兼容层 (C51 关键字 → C++ 桩)
REG52.H      — Keil C51 标准 SFR 寄存器头文件
STARTUP.A51  — Keil 启动汇编 (清零内存 → 跳转 main)
```

## 3. 完整启动流程

```
上电/复位
  │
  ▼
STARTUP.A51 (地址 0x0000)
  ├─ 清零 IDATA (80H 字节)
  ├─ 清零 XDATA (0 字节, 已禁)
  ├─ 设置 SP = ?STACK-1
  └─ LJMP ?C_START (C 运行时初始化)
        │
        ▼
main()  @ main.c:112
  │
  ├─[1] DispInit()                     @ display.c:65
  │     ├─ DispClear(): DispBuf[6] 全写 0x00
  │     ├─ TMOD = 0x11                 ← 两个定时器均设为 16 位模式
  │     ├─ TH1=0xFC, TL1=0x66          ← 1ms 溢出 (11.0592MHz / 12 / 65536-64566 ≈ 1ms)
  │     ├─ ET1 = 1                     ← 开启 Timer1 中断
  │     ├─ EA  = 1                     ← 开启总中断
  │     ├─ TR1 = 1                     ← 启动 Timer1 → T1INTSVC() 开始扫描数码管
  │     ├─ led1 = 1, led2 = 1          ← 初始化灭灯
  │     └─ 返回
  │
  ├─[2] pwmInit()                      @ PWM.c:124
  │     ├─ PWM_EN1=1, PWM_EN2=1        ← 使能 H 桥
  │     ├─ IN1~IN4 = 0                 ← 所有电机停转
  │     ├─ Speed_L=10, Speed_R=10      ← 默认 50% 占空比
  │     └─ 返回
  │
  ├─[3] delay_ms(100)                  ← 等 100ms 让硬件稳定
  │
  ├─[4] 开机自检                        @ main.c:118-124
  │     DispBuf[5..0] = Tab[1..6]      ← 数码管显示 "123456"
  │     delay_ms(2000)                 ← 保持显示 2 秒
  │
  ├─[5] usartInit()                    @ usart.c:8
  │     ├─ SCON = 0x50                 ← 串口模式 1, REN=1 (允许接收)
  │     ├─ TMOD = 0x20                 ← ⚠️ 覆盖 DispInit 的 TMOD!
  │     │                                  Timer1→模式2(8位自动重装, 波特率发生器)
  │     │                                  Timer0→模式0(13位)
  │     ├─ TH1 = 0xFD, TL1 = TH1       ← 9600 bps (SMOD=0, 晶振 11.0592MHz)
  │     ├─ PCON = 0                    ← 波特率不加倍
  │     ├─ TR1 = 1                     ← 启动 Timer1 (现在是波特率发生器)
  │     ├─ EA  = 1                     ← 再次确保总中断开启
  │     └─ 返回
  │
  ├─[6] WIFI_Init()                    @ usart.c:36
  │     发送 6 条 AT 指令配置 ESP8266, 每条间隔 delay:
  │     ┌────────────────────────────────────────────────────────┐
  │     │ 1. "AT\r\n"                        delay_ms(500)       │
  │     │ 2. "AT+CWMODE=1\r\n"              delay_ms(500)       │
  │     │    → STA 模式 (连接路由器)                              │
  │     │ 3. "AT+RST\r\n"                   delay_ms(2000)      │
  │     │    → 重启模块生效                                       │
  │     │ 4. "AT+CWJAP=\"SHYQ-4#-603\",\" \"\r\n"                │
  │     │                                   delay_ms(4000)      │
  │     │    → 连接 WiFi 热点 (密码为空)                          │
  │     │ 5. "AT+CIPSTART=\"TCP\",\"172.25.67.245\",8080\r\n"    │
  │     │                                   delay_ms(2000)      │
  │     │    → 建立 TCP 连接到服务器                              │
  │     │ 6. "AT+CIPMODE=1\r\n"             ────────             │
  │     │ 7. "AT+CIPSEND\r\n"               → 开启透传模式       │
  │     └────────────────────────────────────────────────────────┘
  │     期间通过 led1/led2 亮灭指示每条指令的执行状态
  │
  ├─[7] timer_start_flag = 1             ← 启动 T1INTSVC 中的跑表
  │
  └─[8] while(1) 主循环                   @ main.c:130
        ┌───────────────────────────────────────────┐
        │ led1=1; led2=1;              // 灭灯      │
        │ SENT_At("12345\n");          // 心跳发送  │
        │ buzzer=0; delay_ms(2000);    // 蜂鸣 2 秒 │
        │ buzzer=1;                    // 关蜂鸣    │
        │ ES=1;                        // 开串口中断│
        │ while(key1!=0);              // 等按键    │
        │ motomode++;                  // 切换模式  │
        │ if(motomode>2) motomode=0;   // 0→1→2→0  │
        └───────────────────────────────────────────┘
```

## 4. 中断系统详解

### 4.1 中断向量表 (8051 标准)

| 中断号 | 向量地址 | 中断源 | ISR 函数 | 所在文件 |
|--------|---------|--------|---------|---------|
| 0 | 0x0003 | INT0 外部中断 | (未使用) | — |
| 1 | 0x000B | Timer 0 溢出 | `T0INTSVC()` | main.c:26 |
| 2 | 0x0013 | INT1 外部中断 | (未使用) | — |
| 3 | 0x001B | Timer 1 溢出 | `T1INTSVC()` | display.c:18 |
| 4 | 0x0023 | 串口中断 | `Uart()` | main.c:51 |

### 4.2 Timer 1 ISR — 数码管动态扫描 (display.c:18)

```
T1INTSVC() interrupt (3)
  触发频率: 由 TH1 决定（⚠️ 见下文 Timer 冲突）

  每次中断执行:
  1. TR1=0, 重装载 TH1=0xFC, TL1=0x66, TR1=1    ← 1ms 定时
  2. 消隐: P0=0xFF, CS 锁存 (关闭当前位选)
  3. 段选: P0=DispBuf[n], SS 锁存 (写入段码数据)
  4. 位选: P0=~com[n], CS 锁存 (打开当前位)
  5. n++, if(n>=6) n=0                         ← 循环 6 位数码管

  附加功能 (当 timer_start_flag==1):
  - counter_1ms 累加
  - 每 100 次 (100ms): sec++ (计秒, 到 120 回零)
```

### 4.3 Timer 0 ISR — 电机 PWM + 备用显示扫描 (main.c:26)

```
T0INTSVC() interrupt (1)
  触发频率: ⚠️ Timer 0 从未在 ISR 外部被启动 (无 TR0=1)

  ISR 内部:
  1. TR0=0, 重装载 TH0=0xFC, TL0=0x66, TR0=1    ← 自启动
  2. 数码管扫描 (与 T1INTSVC 逻辑几乎相同)
  3. moto()                                      ← 电机 PWM 状态机
```

### 4.4 UART ISR — 串口指令接收 (main.c:51)

```
Uart() interrupt (4)
  触发条件: RI=1 (接收完成) 或 TI=1 (发送完成)

  ┌─ if(RI) ──────────────────────────────────────┐
  │  RI=0, 读 SBUF → Recive_table[i]              │
  │                                                │
  │  帧同步状态机:                                  │
  │  ┌─ Recive_table[0] == '-' ?                  │
  │  │  YES: i++, led1=0, led2=0 (灯指示)          │
  │  │  NO:  i=0 (帧头错误, 重置)                   │
  │  └──────────────────────────────────────────── │
  │                                                │
  │  if(i >= 9):  一帧收完                         │
  │  ┌─ 校验: [1]=='1' && [2]=='P' && [3]=='d'    │
  │  │  (完整帧头: "-1Pd")                         │
  │  │                                             │
  │  │  通过后:                                     │
  │  │  ├─ 显示 "-1Pd" 到数码管 DispBuf[5..2]      │
  │  │  ├─ switch(Recive_table[7]):               │
  │  │  │   '0'→motomode=0 (停止)                  │
  │  │  │   '1'→motomode=1 (前进)                  │
  │  │  │   '2'→motomode=2 (后退)                  │
  │  │  │   '3'→motomode=3 (左转)                  │
  │  │  │   '4'→motomode=4 (右转)                  │
  │  │  │   default→led1=1 (报错)                  │
  │  │  ├─ [4]=='0' && [5]=='1': led2=0, led1=1   │
  │  │  ├─ [4]=='0' && [5]=='0': led2=1, led1=0   │
  │  │  └─ i=0 (重置索引)                          │
  │  └──────────────────────────────────────────── │
  │                                                │
  └─ else: TI=0 (发送完成, 清除标志) ───────────────┘
```

## 5. 运动控制调用链

```
moto() 由 T0INTSVC 调用 (或被主循环中的某些路径间接调用)
  │
  ├─ motomode=0 (停止)
  │   t=0, IN1~IN4 全低
  │
  ├─ motomode=1 (前进)
  │   t++ (0→19 循环, 周期 20ms)
  │   ├─ t < Speed_L → IN2=1 (左电机正转)
  │   ├─ t < Speed_R → IN3=1 (右电机正转)
  │   └─ 其余时间 IN2=IN3=0 (停转)
  │
  ├─ motomode=2 (后退)
  │   同上周期, 但:
  │   ├─ t < Speed_L → IN1=1 (左电机反转)
  │   └─ t < Speed_R → IN4=1 (右电机反转)
  │
  ├─ motomode=3 (左转)
  │   ├─ 左电机: 同后退 (IN1 反转), PWM 控速
  │   └─ 右电机: IN3=IN4=0 (完全停止)
  │
  └─ motomode=4 (右转)
      ├─ 左电机: IN1=IN2=0 (完全停止)
      └─ 右电机: 同前进 (IN3 正转), PWM 控速
```

## 6. 串口协议详解

### 6.1 指令帧格式 (手机/服务器 → 小车)

```
帧长度: 9 字节
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│ [0] │ [1] │ [2] │ [3] │ [4] │ [5] │ [6] │ [7] │ [8] │
│ '-' │ '1' │ 'P' │ 'd' │  ?  │  ?  │  ?  │ CMD │  ?  │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
  帧头固定 "-1Pd"  (4字节签名)

  [4] = LED 组控制高位 (配合 [5])
        '0' 表示进入 LED 精细控制模式
  [5] = LED 具体控制
        '1' → led2灭, led1亮, DispBuf[0]=Tab[1]
        '0' → led2亮, led1灭, DispBuf[0]=Tab[0]
  [7] = 运动指令 (字符 '0'~'4')
        '0'→停止  '1'→前进  '2'→后退  '3'→左转  '4'→右转
```

### 6.2 心跳发送 (小车 → 服务器)

主循环每轮发送固定字符串: `"12345\n"` (6 字节)

通过 `SENT_At()` 函数逐个字节发送，发送期间 `ES=0` 关串口中断防止冲突。

## 7. ESP8266 WiFi 配网流程

```
MCU (UART AT 指令)              ESP8266
     │                              │
     ├─ "AT\r\n" ─────────────────► ├─ 回应 OK
     ├─ "AT+CWMODE=1\r\n" ────────► ├─ 设为 STA 模式
     ├─ "AT+RST\r\n" ─────────────► ├─ 重启
     ├─ 等 2s                       │
     ├─ "AT+CWJAP=..." ───────────► ├─ 连接 WiFi
     ├─ 等 4s                       │
     ├─ "AT+CIPSTART=..." ────────► ├─ 建立 TCP 连接
     ├─ 等 2s                       │
     ├─ "AT+CIPMODE=1\r\n" ───────► ├─ 开启透传模式
     ├─ "AT+CIPSEND\r\n" ─────────► ├─ 进入透传
     │                              │
     └────────── 透传已建立 ────────┘
     之后所有 SENT_At() 内容直接发往 TCP 服务器
     所有服务器发来的数据通过 UART ISR 接收
```
